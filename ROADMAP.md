# ADP Roadmap

## Role in OASIS
ADP (Architectural Discipline Package) is the quality-and-governance engine for OASIS. It enforces architectural standards across the stack, detects anti-patterns, and produces actionable refactoring guidance so OASIS remains secure, maintainable, and compliant as it scales.

## Current State
- Multi-language analysis (TS/JS, PowerShell, C#, Rust) with statistical scoring and recommendations.
- CLI + ESLint integration with docs, demos, and templates.
- Rich rule catalog, but roadmap content needs consolidation and OASIS alignment.

## Phase 1 — OASIS Quality Baseline (Now → Q2 2026)
- Consolidate roadmap to OASIS-aligned milestones and governance outcomes.
- Publish baseline architectural health metrics for OASIS repos.
- Improve CI integration examples and reporting formats for OASIS compliance.

## Phase 2 — OASIS Governance Automation (Q3–Q4 2026)
- Adopt Praxis for pattern detection, refactoring policy, and compliance gates.
- Add architecture trend tracking with local baselines and regression alerts.
- Expand SARIF output and security toolchain integration.

## Phase 3 — OASIS Scale & Ecosystem (2027)
- IDE real-time feedback (VS Code/JetBrains) for OASIS teams.
- Broader language support and plugin analyzers for ecosystem extensions.
- Fleet-wide architecture dashboards and portfolio-level governance views.
